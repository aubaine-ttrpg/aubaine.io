# Codex

The mechanics lab and source of truth for Aubaine. The rules, their numbers, and the math that
produces those numbers live here together, in English.

## Domains

The codex is organised into domains, one folder per purpose. A domain holds its rules as
Markdown, a `lab/` of Python that derives and checks its numbers, and a `data/` of JSON the lab
reads and writes.

## The lab and data contract

- Some `data/` files are authored (design inputs); the rest are generated by the lab and
  committed, so the reasoning stays visible in git.
- Domains share numbers only through `data/` JSON, never by importing each other's Python.
- Generated files are reproducible: `make regen` remakes them byte for byte, so no number can
  drift from the formula that made it.

## Rule shape

A rule reads on its own: what it is, how it works, its outcomes, and a short play example.
Anything about why it works lives in a separated `Design Notes` section at the end and is not
part of the rule.

## Running the lab

    make install   # create .venv and install dev tools
    make test      # run the balancing-lab tests
    make lint      # ruff
    make regen     # regenerate every domain's data files
