# Archive — `templates/` (Twig)

v1 Twig templates. Server-rendered UI, enhanced by Stimulus controllers (archived under `_archive/assets/controllers/`) and Turbo.

## Layout

- `base.html.twig` — root layout: `<head>`, asset imports (via Webpack Encore's `encore_entry_link_tags` / `encore_entry_script_tags`), Turbo + Stimulus bootstrap, global flash region.

## `admin/` — dev-gated CRUD UI

Each domain gets its own sub-folder with `index`, `new`/`edit`, `show`, and a shared `_form.html.twig` partial.

- `index.html.twig` — admin landing page
- `skill/` — Skills CRUD + export (`export_form`, `export_result`) + a `help.html.twig` onboarding page + `_form_fields.html.twig` extracted partial
- `skill_tree/` — Skill tree CRUD + a dedicated `export.html.twig`
- `tag/` — Tags CRUD
- `old_skill/` — Legacy skills CRUD + export (drop in v2)
- `factory/` — Form-based procedural generators (`skill.html.twig`, `old_skill.html.twig`)
- `character_sheet/` — (empty placeholder; the sheet UI was never built in v1)
- `components/` — (empty placeholder)

## `components/` — Twig Component templates

Used via `UXTwigComponent` (`<twig:SkillPlate .../>` style syntax). Paired with PHP classes in `_archive/src/Twig/Component/` where needed.

- `SkillPlate.html.twig` — the visual skill card (pairs with `_archive/assets/styles/skill-plate.css`)
- `SkillExportCard.html.twig` — printable export layout for a Skill (backed by `src/Twig/Component/SkillExportCard.php`)
- `AnonSkillExportCard.html.twig` — variant for anonymous/unnamed skills (used by the inline-create flow in the tree builder)
- `alerts.html.twig` — flash message rendering (pairs with `auto_dismiss_controller.js`)
- `content_nav.html.twig` — in-page nav (pairs with `content_nav_controller.js`)

## `dev/` — sandbox / builder

- `skill_tree_builder.html.twig` — hosts the 927-LOC `skill_tree_builder_controller.js`
- `grid_test.html.twig` — throwaway grid-layout experimentation

## `public/` — empty

Was reserved for player-facing pages. Never populated in v1.

## What to pull forward

- **Pattern, not code:** the `_form.html.twig` partial pattern (shared by `new` + `edit`) is a clean Symfony idiom worth preserving.
- **Twig components:** `SkillPlate` and `SkillExportCard` show a good pattern for reusable rendering, if UXTwigComponent is added to v2.
- **Drop:** everything under `admin/old_skill/`, `admin/factory/`, and the empty `character_sheet/` / `public/` / `admin/components/` folders.
- **Rewrite:** all admin CRUD templates will be re-generated by `make:crud` against the v2 schema anyway.
