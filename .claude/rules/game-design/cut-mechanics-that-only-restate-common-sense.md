# Cut mechanics that only restate common sense

**Rule:** Every mechanic must earn its cost in reading and table time.

- If a rule only says what the fiction already implies, delete it. A rule that tells the GM to "decide what makes sense" adds a page and changes nothing.
- Add a mechanic only when it creates a choice, a cost, pacing, tension, genre feel, or fair resolution. If it does none of these, cut it.
- Weigh it against the action economy: a new option competes for the same turn as every other option, so it must be worth spending a turn on. Weigh it against the reward loop: it should push the playstyle the system rewards, not pull against it.

**Why:** Rules add cognitive load, so the aubaine-content-writer skill's check is to cut a rule that only restates common sense (`docs/best-practices.md`, "Avoid Unnecessary Mechanics"), and to respect the action economy and fit the system's reward loop ("Respect Action Economy", "Fit the System's Reward Loop"). Its Mechanics Gate checks that existing options are not overshadowed (`checklists/quality-gates.md`). A redundant rule slows every table that reads it while adding nothing to play.

**Good / Bad:**
```markdown
Bad:
Talking Rule: characters may speak to each other during their turn. Speaking
takes no action. The GM decides what an NPC says back.

Good:
(no rule; free talk needs none. Add a rule here only if speaking costs an
action, opens a reaction, or feeds a social clock.)
```
