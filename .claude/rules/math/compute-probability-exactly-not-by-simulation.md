# Compute probability exactly, not by simulation

**Rule:** Derive every probability by **full enumeration weighted with `fractions.Fraction`**, so the result is an exact rational. Enumerate the outcomes (or the sorted multisets, each weighted by its multiplicity), sum the integer counts, and divide by the integer total. Never sample: no Monte Carlo, no `random` module, no `numpy.random`, no "run N trials and take the ratio". A returned probability is a `Fraction` (or an integer count), never an estimate.

**Why:** A sampled probability carries sampling error and lands on different digits every run, so it can neither be asserted exactly nor regenerated byte for byte. Full enumeration over `Fraction` yields the true value with no error term, and the counts for `n` dice sum to `faces ** n` by construction, an identity the code can assert against. The codex is the exact-math balancing lab (codex/README.md), and its resolution `dice.py` builds every pmf this way. Python's `fractions.Fraction` holds an exact ratio of integers as a member of the numeric tower (PEP 3141).

**Good / Bad:**
```python
# Bad: Monte Carlo. Noisy float, a different answer every run, never exact.
import random

def p_at_least(target: int, n: int, faces: int, trials: int = 1_000_000) -> float:
    hits = sum(1 for _ in range(trials) if sum(random.randint(1, faces) for _ in range(n)) >= target)
    return hits / trials
```
```python
# Good: enumerate the multisets, weight by multiplicity, divide exact integer counts.
from fractions import Fraction
from itertools import combinations_with_replacement

def p_at_least(target: int, n: int, faces: int) -> Fraction:
    total = Fraction(faces**n)
    hits = sum(
        _multiplicity(combo)
        for combo in combinations_with_replacement(range(1, faces + 1), n)
        if sum(combo) >= target
    )
    return hits / total  # exact rational, identical on every machine and run
```
