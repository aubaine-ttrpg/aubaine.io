# Make generation reproducible

**Rule:**

- Every generated `data/` file is **byte-for-byte reproducible**: `make regen` remakes it identically, so re-running with no source change leaves git clean.
- Every generated file carries a **provenance stamp** naming its generator, a top comment `<!-- generated by <domain>/lab/generate.py -->` for text and Markdown, or the equivalent one-line stamp key for a format that has no comments. A reader must be able to tell the file is derived and where it came from. Never hand-edit a generated file.
- **Round every float to a fixed precision** before writing it, so the same computation produces the same bytes on any machine or Python build. Do the rounding in the generation layer, not in the math.

**Why:** A generated file that cannot be reproduced becomes a second source of truth that drifts from the formula that made it. Byte-stable output lets a regenerate-and-compare test prove the committed file still matches its generator (codex/README.md: generated files are reproducible, `make regen` remakes them byte for byte). A provenance stamp stops a future reader or agent from editing a derived file by hand. Fixed-precision rounding removes the last-digit float noise that would otherwise make one computation emit different bytes.

**Good / Bad:**
```python
# Bad: no stamp, and the raw float's last digits differ between machines.
def main() -> None:
    write_text(DATA / "rates.md", f"P(success) = {float(p_success(20))}")
```
```python
# Good: stamp names the generator, fixed precision gives stable bytes, regen stays clean.
ROUND_PLACES = 6
STAMP = "<!-- generated by dice-and-modifiers/lab/generate.py -->\n\n"

def main() -> None:
    p = round(float(p_success(20)), ROUND_PLACES)  # fixed precision -> identical bytes everywhere
    write_text(DATA / "rates.md", f"{STAMP}P(success) = {p}")
```
