# Bookcraft and layout standards for tabletop RPG visual supports

## The book is a play tool

A TTRPG book is not just a container for lore. It is a live-use interface. Every layout decision should help one of these reader modes:

- **Learning mode:** A new player learns the game in order.
- **Prep mode:** A GM prepares scenes, locations, NPCs, clocks, factions, treasure, and consequences.
- **Table mode:** Players search quickly during play.
- **Reference mode:** The group revisits exact rules and tables after learning them.

## Grid and page architecture

Create a page system before styling:

- Trim and bleed.
- Margin/gutter model.
- Column count and gutter width.
- Baseline grid.
- Heading hierarchy.
- Sidebar/callout widths.
- Table styles.
- Example blocks.
- Stat block templates.
- Footer/running head/folio placement.

TTRPG books often need two related grids: a readable prose grid and a dense reference grid. Do not force long rules explanations into a stat-block grid.

## Hierarchy

A page should reveal its structure at a glance:

1. Chapter/section.
2. Current procedure or location.
3. What the table needs now.
4. Examples, exceptions, and designer notes.
5. Cross-references.

Use consistent signals: size, weight, spacing, rules, icons, labels, and placement. Do not invent a new style for every sidebar.

## Typography

- Set body measure around 45-75 characters per line for prose.
- Shorter measures are acceptable for tables, sidebars, and handouts.
- Use tabular figures for tables, dice ranges, stat blocks, page references, and costs.
- Avoid faux small caps; use real small caps if available.
- Use bold for game terms and rule triggers, not decorative emphasis.
- Use italics sparingly; avoid italic paragraphs for rules.
- Keep heading jumps meaningful; do not skip from H2 to H4 unless the source format requires it.

## Tables

TTRPG tables are gameplay engines, not decoration.

- Put die range in the first column.
- Name the table’s purpose in the title: “d6 Signs the Cult Was Here,” not “Atmosphere.”
- Keep results actionable.
- Use repeated header rows for tables that cross pages.
- Avoid table cells that continue for more than a few lines; split into sub-tables or procedures.
- Use adequate row height for home printing and annotation.
- Do not rely on zebra striping alone; maintain rules/borders and contrast.

## Sidebars and callouts

Use sidebars for material that helps without interrupting the core procedure:

- Examples.
- Optional rules.
- Designer notes.
- Safety guidance.
- Conversion notes.
- Quick references.

Do not hide mandatory procedures in sidebars unless the sidebar is visually part of the procedure.

## Front matter

Best-practice front matter for book-like publication artifacts often includes:

- Half title, if desired.
- Title page.
- Credits and copyright page.
- License and third-party attribution.
- Compatibility disclaimer.
- Safety and accessibility note, when relevant.
- Table of contents.
- “How to use this book.”
- Quickstart or first-session route.

Book Design Made Simple notes that title and copyright pages are core front matter; TTRPGs often add license, safety, accessibility, and quickstart pages because they are play tools. See citation 27.

## Back matter

Consider:

- Index.
- Glossary.
- Condition summary.
- Tables appendix.
- Character sheet.
- License notices.
- Asset credits.
- Change log/errata URL.
- Production colophon.
- Blank notes pages only if useful for table play, annotation, or print editions.

## Format-specific support details

### Booklets and zines

- Page count must resolve to a multiple of 4 for saddle stitching.
- Keep central spread/gutter safe.
- Avoid important details near fold creep on thick signatures.
- Home-print versions should have low ink coverage and no essential full-bleed areas.

### GM screens

- Design panel-by-panel, not as a mini-book.
- Prioritize tables used during play: conditions, prices, encounter loops, clocks, travel, damage, NPC moves.
- Keep font sizes larger than book text; the screen is read at distance.

### Character sheets

- Use pencil-friendly fields.
- Keep labels outside writing areas when possible.
- Make repeated calculation fields explicit.
- Provide grayscale-friendly design.
- Do not use decorative texture behind fill fields.

### Reference cards

- One concept per card.
- Stable title position.
- High contrast.
- Minimum type size appropriate to card size.
- Leave room for bleed and corner radius if cards are cut.
