# Workflow: GM screen

## Screen design principle

A GM screen is a distance-readable dashboard, not a compressed rulebook.

## Panel inventory

Prioritize the most frequently referenced procedures:

- Core resolution.
- Conditions/status effects.
- Damage/harm/recovery.
- NPC/monster actions.
- Travel/exploration turns.
- Encounter/reaction tables.
- Prices/equipment.
- Difficulty benchmarks.
- Clocks/fronts/faction moves.
- Safety and pacing reminders.

## Layout rules

- Use larger type than book text.
- One panel = one functional cluster when possible.
- Avoid long paragraphs.
- Use tables, lists, and procedure boxes.
- Keep color contrast strong.
- Do not place essential information near folds, panel edges, or mounting seams.

## Export

- Confirm panel size with the printer or screen insert product.
- Include bleed/safety for each panel.
- Export outside/player-facing and inside/GM-facing files separately.
