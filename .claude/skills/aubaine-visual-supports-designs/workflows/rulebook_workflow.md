# Workflow: rulebook or core book

## 1. Define the reading paths

Create three paths:

- **Learn path:** what a first-time reader needs in order.
- **Play path:** what the group needs during a session.
- **Reference path:** where exact rules live after learning.

## 2. Recommended chapter sequence

Adapt to the game, but start from this:

1. What this game is.
2. What you need to play.
3. Safety, tone, and table expectations.
4. Core mechanic.
5. Characters / roles / playbooks.
6. Basic procedures of play.
7. Conflict, danger, harm, recovery, advancement.
8. GM procedures.
9. Setting tools, factions, locations, threats.
10. Bestiary/NPCs/items/spells, if applicable.
11. Tables and references.
12. Sheets, glossary, index, license, credits.

## 3. Rule page template

For each rule subsystem:

- One-sentence purpose.
- Trigger.
- Step-by-step procedure.
- Player choices.
- GM choices.
- Outcomes.
- Example.
- Exceptions.
- Cross-reference.

## 4. Layout pass

- Use prose grid for explanation chapters.
- Use reference grid for tables and stat blocks.
- Place examples near the rules they demonstrate.
- Use running heads by chapter and section.
- Use consistent page anchors for common subsystems.

## 5. Output pass

- Define screen PDF, accessible PDF, printer PDF, home-print PDF, and source package needs.
- For digital editions, confirm bookmarks, links, page labels, metadata, and searchable text.
- For print editions, confirm trim, binding, page count, bleed, color mode, output intent, and cover template.
- Run contrast and manuscript lint before layout lock.
- Run export preflight after PDF export.
