# Workflow: adventure module

## 1. Situation, not script

Build a module around pressures and factions, not a predetermined plot.

Required module spine:

- Premise.
- Hook menu.
- Stakes.
- Countdown/escalation.
- Locations or nodes.
- Factions and NPC motives.
- Discoverable clues.
- Threats and consequences.
- Rewards and costs.
- Exit states.

## 2. Opening spread

The first spread should let a GM run the premise tonight:

- One-paragraph pitch.
- Content warning/safety note if applicable.
- Level/rank/player count/runtime.
- Required rules references.
- Map or node diagram.
- What happens if the players do nothing.

## 3. Keyed locations

Each keyed entry should contain:

- Immediate sensory fact.
- Interactive objects.
- Occupants and what they are doing now.
- Clues/secrets.
- Danger/complication.
- Treasure/resource/change.
- Connections/exits.

## 4. NPC and faction pages

For each faction/NPC:

- Want.
- Fear.
- Leverage.
- First move.
- Escalation move.
- What changes their mind.
- What they know.

## 5. Tables

Include table-useful generators:

- Rumors.
- Encounters.
- Signs of danger.
- Complications.
- Rewards.
- Time pressure.
- NPC reactions.

## 6. Layout pass

- Keep map references visible near keys.
- Use page references in encounter tables.
- Repeat stat block field order.
- Do not bury required clue chains in prose.
- Add a one-page GM reference if module exceeds 16 pages.
