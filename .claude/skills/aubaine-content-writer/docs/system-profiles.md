# System Profiles

Use this file to match output to a game's design language.

## System-Neutral Default

Use when no game system is named.

Write with:

- Fiction-first triggers
- Clear costs
- Plain difficulty labels
- Simple resources
- Optional clocks
- Adaptation notes

Avoid assuming:

- Hit points
- Armor class
- Spell slots
- Classes
- Levels
- Challenge ratings
- Skill lists

Default terms:

- Easy
- Risky
- Hard
- Minor cost
- Major cost
- Harm
- Stress
- Supply
- Favor
- Heat
- Clock
- Advantage
- Setback

## d20 Fantasy / 5e-Compatible

Use when the user asks for D&D-like fantasy.

Respect:

- Action economy
- Save DCs
- Attack bonuses
- Armor class
- Hit points
- Bounded accuracy
- Rest resources
- Spell concentration
- Legendary or lair actions for solo threats
- Encounter objectives beyond reducing HP

Design tips:

- Give solo monsters bonus actions, reactions, legendary-style actions, terrain, phases, or minions.
- Add objectives: stop ritual, rescue hostage, hold bridge, steal relic, close portal.
- Telegraph high-damage effects.
- Avoid save-or-sit-out effects unless short, avoidable, or counterplay exists.
- Include noncombat resolutions when possible.

Stat block guidance:

- Use original names and abilities.
- Do not copy closed-source stat blocks.
- If using SRD material, cite and comply with the relevant license.

## Pathfinder 2e-Compatible

Respect:

- Encounter XP budgets
- Creature levels relative to party level
- Three-action economy
- Degrees of success
- Conditions
- Hazards
- Tight math

Design tips:

- Do not casually inflate modifiers.
- Terrain, hazards, and objectives should matter.
- Solo severe threats are dangerous; use the official difficulty frame.
- Include skill actions and alternate routes.

## OSR / NSR

Respect:

- Rulings over rules
- Player skill
- Exploration procedure
- Resource pressure
- Morale
- Reaction rolls
- High lethality
- Fast stat blocks
- Treasure as incentive
- Open-ended problem solving

Design tips:

- Telegraph danger clearly.
- Make traps detectable through description.
- Reward mapping, caution, negotiation, and equipment use.
- Use factions and wandering encounters.
- Avoid balanced set-piece assumptions.
- Make treasure risky, bulky, hidden, or faction-relevant.

## Powered by the Apocalypse

Respect:

- Fictional triggers
- GM agenda and principles
- GM moves
- Partial success
- Play to find out
- Moves that snowball

Design tips:

- Do not write plot outcomes.
- Write threats, fronts, impulses, stakes questions, and custom moves.
- Use 10+/7-9/6- style outcomes when appropriate.
- On a miss, make a GM move that follows from fiction.
- Custom moves must have clear fictional triggers.

## Forged in the Dark

Respect:

- Position
- Effect
- Clocks
- Stress
- Resistance
- Consequences
- Load
- Flashbacks
- Heat
- Faction status
- Scores and downtime

Design tips:

- Define obstacles, not methods.
- Use clocks for complex pressure.
- Make consequences concrete: harm, reduced effect, worse position, lost opportunity, clock ticks, collateral damage, heat.
- Keep score plans flexible.
- Reward bold action and messy fallout.

## GUMSHOE / Investigation-First

Respect:

- Core clues must be found
- The challenge is interpretation
- Point spends provide extra benefit, speed, leverage, or detail

Design tips:

- Do not require a roll to find required clues.
- Use spends or risks for extra context, not basic progress.
- Give clues multiple meanings and useful false leads without dead ends.

## Horror

Respect:

- Vulnerability
- Uncertainty
- Limited resources
- Escalation
- Consent
- Mood through concrete detail

Design tips:

- Use restraint.
- Show aftermath before threat.
- Make safety tools visible.
- Avoid graphic harm unless requested and appropriate.
- Use costs, isolation, compromised gear, unreliable witnesses, and moral pressure.

## Mystery

Respect:

- Fair clues
- Redundancy
- Active suspects
- Timelines
- Motives
- Evidence chains

Design tips:

- Give each suspect means, motive, opportunity, alibi, and pressure point.
- Make wrong theories create complications, not dead ends.
- Keep a truth map for the GM.

## Political / Intrigue

Respect:

- Leverage
- Public face vs. private motive
- Reputation
- Debt
- Favors
- Information asymmetry

Design tips:

- Every faction wants something now.
- Every NPC has a price, fear, or line they will not cross.
- Make consequences social, legal, financial, and relational.

## Solo / Journaling

Respect:

- Prompts
- Procedures
- Reflection
- Escalating constraints
- Clear end states

Design tips:

- Write prompts that require choice, memory, contradiction, or change.
- Give procedures for when to roll, write, mark, or end.

## Rules-Light / One-Page Games

Respect:

- Minimal cognitive load
- Strong procedure
- Few stats
- Fast resolution
- Direct examples

Design tips:

- Cut edge cases.
- Use a single core mechanic.
- Make every rule earn its space.
