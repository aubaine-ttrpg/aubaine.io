# Docs

Technical documentation for developers working on this repo. Each page is a topic-oriented architecture overview: what the codebase looks like and why it was built that way.

New pages follow [Rule 08 — Technical Documentation](../rules/08-technical-documentation.md): `UPPER_SNAKE_CASE.md` filename, topic as the slug, and a structure of Overview / Decisions (with rejected alternatives) / Verification / optional Follow-ups.
